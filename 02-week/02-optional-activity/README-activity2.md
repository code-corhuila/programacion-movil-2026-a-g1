# Planificación Ágil del Proyecto – IceCream-App

**Asignatura:** Programación Móvil  
**Proyecto:** IceCream-App  
**Metodología:** Scrum

---

# 1. Elección de Metodología

Se selecciona **Scrum** como una metodología ágil para el desarrollo de IceCream-App.

**Justificación:**

1. **Desarrollo incremental del MVP:**  
   La aplicación se construirá por funcionalidades (menú, personalización, carrito, pedidos), lo cual se adapta al enfoque por sprints de Scrum.

2. **Priorización clara del backlog:**  
   El proyecto tiene funcionalidades Must/Should/Could, lo que encaja naturalmente con la gestión de backlog y planificación de sprints.

3. **Seguimiento y control del progreso:**  
   Scrum permite revisar avances semanales del MVP, reduciendo riesgos técnicos y de alcance en el desarrollo móvil.

# 6. Definición de Terminado (DoD)

✅ La funcionalidad compila sin errores  
✅ Navegación completa de la historia  
✅ Datos mostrados correctamente  
✅ Lógica funcional validada  
✅ Código subido al repositorio  
✅ Evidencia (captura o video)  
✅ Historia movida a Done
